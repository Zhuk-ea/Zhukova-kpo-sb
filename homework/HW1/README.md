# ERP-система Московского зоопарка

Данное приложение представляет из себя систему автоматизации учета животных и инвентаря для Московского зоопарка.

## Реализованный функционал

- **Учет животных** - добавление с проверкой здоровья через ветеринарную клинику
- **Инвентаризация** - уникальные номера для животных и вещей
- **Контактный зоопарк** - автоматический отбор подходящих животных
- **Расчет питания** - общая потребность в еде для всех животных
- **Управление** - добавление, удаление, просмотр инвентаря

## Применение принципов SOLID

### **S - Single Responsibility (Принцип единственной ответственности)**
- `Zoo` - управление зоопарком (добавление, удаление, инвентаризация)
- `VeterinaryClinic` - проверка здоровья животных
- `Animal` и его наследники - представление животных с их специфической логикой
- `Thing` и наследники - представление инвентаря

### **O - Open/Closed (Принцип открытости/закрытости)**

Благодаря структуре классов, легко добавить новый вид животного, учёт сотрудников и т.д

### **L - Liskov Substitution (Принцип подстановки Лисков)**
Любой наследник Animal может быть использован вместо базового класса
```java
Animal rabbit = new Rabbit("Кролик", 1, 7);
Animal tiger = new Tiger("Тигр", 10);
Animal monkey = new Monkey("Обезьяна", 3, 6);
```
Все они работают одинаково через общий интерфейс
```java
zoo.addAnimal(rabbit);
zoo.addAnimal(tiger);
zoo.addAnimal(monkey);
```

### **I - Interface Segregation (Принцип разделения интерфейсов)**

- Alive - только для живых существ (getFood(), setFood())
- Inventory - только для инвентаризации (getNumber(), setNumber())
- Животные реализуют оба интерфейса, вещи - только Inventory

### **D - Dependency Inversion (Принцип инверсии зависимостей)**

Классы зависят от интерфейсов, а не от конкретных реализаций

## Инструкции по запуску
### Требования
- Java 21
- Gradle 8.0+
### Старт
```bash
# Клонирование репозитория
git clone <repository-url>
cd zoo-erp-system
# Запуск приложения (автоматическая демонстрация работы)
./gradlew bootRun
```
### Полная проверка проекта

```bash
    # Очистка, сборка, тесты, проверка checkstyle
./gradlew clean build

# Запуск только тестов
./gradlew test

# Просмотр отчетов
open build/reports/jacoco/test/html/index.html  # Покрытие тестами
open build/reports/tests/test/index.html        # Результаты тестов
open build/reports/checkstyle/main.html         # Проверка стиля
```
